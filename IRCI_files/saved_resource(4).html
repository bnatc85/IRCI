IRCI: Methodology

Quarter-level, peer-relative percentiles (0–100), then a weighted composite.

1) Mathematical Definitions

Notation

  • i = ticker (company), q = calendar quarter, t = trading day, j = news item.
  • Pq = peer set in quarter q (e.g., the symbols provided).
  • percq(⋅) = empirical percentile across Pq in quarter q (0–100).
  • Weights (defaults shown): Composite: wv=0.35,wℓ=0.35,wc=0.15,ws=0.15 Trust: we=0.50,wb=0.20,wm=0.30
  • All quarter buckets are the UTC quarter end at 00:00:00.

Valuation Dial

Compute the multiple and a peer-relative gap, then percentile the “cheapness.”

EV/EBITDA multiple: Mi,q=EVi,qEBITDAi,qTTM Peer mean (ex-self): μq(−i)=1|Pq|−1∑k∈Pq∖{i}Mk,q Relative gap: gi,q=Mi,q−μq(−i)μq(−i) Percentile (lower multiple ⇢ higher score): Vi,q=percq(−gi,q)(equivalently, percq(−Mi,q))

Liquidity Dial

Blend turnover, Amihud illiquidity, and Roll spread (all quarter summaries). Higher = more liquid.

Daily Amihud (per day t): Ai,t=|ri,t|DollarVolumei,t Quarter statistic (e.g., median): A¯i,q=mediant∈qAi,t Daily turnover (shares): Ti,t=VolumeSharesi,tSharesOutstandingi,t⇒T¯i,q=mediant∈qTi,t Roll (effective) spread (Roll, 1984): Si,qRoll=2−Cov(Δpi,t,Δpi,t−1) (optionally converted to bps).

Percentiles (directional): pi,qturn=percq(T¯i,q),pi,qami=percq(−A¯i,q),pi,qsprd=percq(−Si,qRoll) Liquidity score (weights renormalized if a term is missing): Li,q=100⋅wTpi,qturn+wApi,qami+wSpi,qsprdwT+wA+wS

Coverage Dial

Tracks the cadence of 8-K updates and timeliness to the 10-Q/K after the quarter end.

Quarter 8-K count: Ki,q=#{Form 8-K for i with event date in q} Timeliness (days from quarter end to the first 10-Q/K filing that covers that period): Di,q=days(quarter\_endq→first 10-Q/K) Percentiles: pi,q8K=percq(Ki,q),pi,qtime=percq(−Di,q) Coverage score: Ci,q=100⋅w8Kpi,q8K+wtimepi,qtimew8K+wtime

Trust (Sentiment/Calmness) Dial

Combines residual volatility “calmness,” calmness around SEC events, and media tone.

  1. Baseline calmness via factor model residuals. Regress excess returns on Fama-French 5 (or CAPM fallback), take residuals εi,t. BaselineRawi,q=−stdevt∈q(εi,t) pi,qbase=percq(BaselineRawi,q)
  2. Event calmness around SEC events d∈Di,q (8-K and 10-Q/K). With a k-day window (default k=3): S(d)=∑u=−kk|εi,d+u|,EventRawi,q=−mediand∈Di,qS(d) pi,qevent=percq(EventRawi,q)
  3. Media tone from news within q (FinBERT; VADER fallback). If si,j is the per-article sentiment (−1..+1), use the mean: ToneRawi,q=1Ni,q∑j=1Ni,qsi,j,pi,qtone=percq(ToneRawi,q)

Trust (a.k.a. sentiment) percentile (weights renormalized if any piece is missing):

Si,q=100⋅wepi,qevent+wbpi,qbase+wmpi,qtonewe+wb+wm

Composite IRCI

Weighted average of available dial percentiles (0–100), row-wise weight renormalization if any dial is missing:

IRCIi,q=wvVi,q+wℓLi,q+wcCi,q+wsSi,q∑weights of available dials

Within-quarter rank is computed by sorting IRCIi,q descending (1 = best).

2) Metrics & Features Used

Valuation

  • Enterprise Value (EV): market cap + total debt − cash & equivalents.
  • EBITDA (TTM): operating income ± D&A fallback if needed (as in code).
  • EV/EBITDA multiple (Mi,q).
  • Peer mean (ex-self) of EV/EBITDA.
  • Relative gap gi,q and percentile mapping to Vi,q.
  • Metadata: as-of date, quarter bucket, peer count.

Liquidity

  • Daily OHLCV (adj close, volume).
  • Dollar volume: price × volume.
  • Amihud illiquidity: |r|/DollarVolume; quarter median.
  • Turnover: shares traded / shares outstanding (median), if available.
  • Roll spread (effective spread estimator), optionally in bps.
  • Percentiles pturn,pami,psprd and blended Li,q.

Coverage

  • SEC 8-K filings: count per quarter.
  • SEC 10-Q/10-K: days from quarter end to first filing (period covered).
  • Percentiles for 8-K cadence and timeliness, combined into Ci,q.
  • Filing metadata used: CIK, acceptedDate, filingDate, periodOfReport, form type.

Trust / Sentiment

  • Daily returns & factor model residuals (FF5 or CAPM fallback).
  • Baseline calmness: stdev of residuals (negated).
  • Event calmness: median sum of |residuals| in ±k-day windows around 8-K and 10-Q/K dates (negated).
  • Media tone: mean article sentiment in quarter; article count recorded.
  • Percentiles pevent,pbase,ptone blended to Si,q.

3) Data Sources & Fallbacks

Prices & Market Data

  • Primary: FinancialModelingPrep (FMP) historical prices (OHLCV).
  • Fallback: yfinance (Yahoo Finance) for prices when FMP is rate-limited.
  • Benchmark (fallback CAPM): SPY via yfinance.

Factors

  • Primary: Fama-French 5 daily factors (Ken French Data Library).
  • Fallback: CAPM using market proxy (e.g., SPY) if FF5 unavailable.

SEC Filings

  • Primary: SEC EDGAR JSON/REST endpoints (CIK map, submissions; 8-K, 10-Q/10-K metadata).
  • Company mapping: SEC “company_tickers” JSON.
  • Headers: custom User-Agent (per SEC guidance).

Fundamentals

  • Primary: FMP endpoints for EV & EBITDA (TTM), with fallbacks to operating income + D&A logic in the code path.

News & Sentiment

  • Primary news ingestion: CSV supplied via --news-csv (date, ticker, title/text).
  • Optional: FMP News (if enabled) or any aggregator you export to CSV.
  • Sentiment model (primary): FinBERT (finance-domain BERT).
  • Fallback: VADER (nltk.sentiment) if FinBERT not available.

Missing data: each dial renormalizes weights row-by-row so a missing piece (e.g., no news) is dropped without penalizing other components.
